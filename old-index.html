In 2010 I had an internship in the ROSE Compiler group at the Lawrence Livermore National Laboratory. I learned about source-to-source compilation and its potential. I now focus my PhD work on extending the scope of this technology.
The question that intrigate me is: To what extend can, a source-to-source compiler, be integrated in the workflow of a software engineering team?.

Research

Generation of Optimized Computation Kernels

Most applications spend most of there time in loops. Researches on how to optimize these compute intensive regions have been ongoing for many years. Our approach look at the diversity of computation kernels that can be generated for a given region.

For this research, we use OpenACC codes as input and generate OpenCL C Kernels. The first issue we address is how the compiler can exploit information provided by the develloper to generate many different computation kernels. The second issue is to find which computation kernels need to be use in a given situation (including target hardware and workload).

Directives for Source-to-Source Compiler

Compiler directive are extremely versatile. They range from simple commands (ie. unroll directive) to complete programing model (ie. OpenMP). Specifically, we are looking at how compiler directives can be leveraged in a source-to-source environment.
